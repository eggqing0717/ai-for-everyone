## 一、先决条件（能不能通过审核）
Google AdSense 有审核门槛，你项目已具备大部分：
- ✅ **自定义域名**（AdSense 基本要求，`github.io` 子域一般不收）——你要先把域名绑上；
- ✅ **隐私政策页 / 关于页**（已有 `privacy-policy.md`、`about.md`，AdSense 必需）；
- ⚠️ **原创内容要充足**：建议正文配图补齐、内容再丰富些（审核偏好"有实质内容的成熟站点"）。

## 二、申请拿到发布商 ID
1. 用 Google 账号注册 [AdSense](https://adsense.google.com)，填入你的网站域名；
2. 注册后会拿到一个**发布商 ID**：`ca-pub-XXXXXXXXXXXXXXXX`；
3. Google 要求你把一段 `<head>` 脚本放到站点所有页面上，然后它来审核。

## 三、放广告需要做两件事
**1️⃣ `ads.txt`**（根目录，声明你是合法发布商）
把项目里的 `ads.txt` 填成：
```
google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0
```

**2️⃣ 在每个页面 `<head>` 注入 AdSense 加载脚本**
```html
<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXXXXXXXX" crossorigin="anonymous"></script>
```

## 四、推荐的稳妥注入方式：构建后注入（不碰模板）
既然改模板会让构建失败，最稳的做法是**在 GitHub Actions 里，`npm run build` 之后、部署之前，加一步把上面那段脚本自动插进 `_book/` 里所有 HTML 的 `<head>`**。这样：
- 不改 Honkit 模板 → 构建绝不会坏；
- 全站所有页面自动带上广告脚本；
- `ads.txt` 也一并复制进 `_book`。
