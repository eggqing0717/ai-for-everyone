# 第22章 RAG 检索增强：给 AI 装上"长期记忆"

你有没有发现，AI 有时会一本正经地胡说八道，或者答不上你公司内部的事？RAG 就是专治这两个毛病的良方。

## 为什么需要 RAG

大模型虽然聪明，但有两个天生的短板：

- **知识会过时**。它的知识"截止"在训练的那一刻。你问它昨天的新闻、今年的新政策，它一无所知。
- **会瞎编（幻觉）**。遇到不知道的事，它不会说"我不知道"，反而会一本正经地编一个看起来很像真的答案。
- **不懂你的私有资料**。你公司的产品手册、内部规章、客户档案，它从没见过，自然答不上来。

怎么办？总不能为了每天的新消息就重新训练一遍模型吧（那太贵了）。于是聪明人想出一招：**答题前，先让它查资料。**

这就是 **RAG**（检索增强生成，Retrieval-Augmented Generation）——**先检索、再回答**。

## 一个生活化的比喻：开卷考试

闭卷考试时，你只能凭脑子里记住的东西答题，记不清就容易瞎蒙——这就是普通大模型。

而 RAG 就像**开卷考试**：答题前，你先翻开资料书，找到相关那几页，看着资料再作答。答案自然又准又有依据。

RAG 给 AI 配的这本"随时可查的资料书"，就相当于给它装上了一个**外挂的长期记忆**。资料随时能更新，AI 本身却不用重新训练。（这只是类比，实际更复杂。）

## RAG 是怎么工作的

整个流程可以拆成简单的几步：

1. **建资料库**：事先把你的文档（手册、条款、百科……）切成一小段一小段，存进一个特殊的数据库。
2. **理解提问**：你提出问题，比如"我们公司的退货政策是几天？"
3. **检索相关资料**：系统在资料库里，找出和你问题**最相关**的那几段内容。
4. **拼进提问**：把找到的资料和你的问题拼在一起，变成一段"带资料的提问"喂给大模型。
5. **基于资料作答**：大模型**看着资料**来回答，还能标注答案出自哪份文件。

一句话总结：**检索相关文档 → 拼进提问 → 让模型基于资料作答。**

## 关键角色：向量数据库

上面第 3 步"找出最相关的资料"，是怎么做到的？靠的是**向量数据库**。

还记得前面讲过的"词向量"吗——文字可以被转成一串数字（向量），**意思相近的内容，数字也相近**。RAG 把每一段资料都转成这样一串数字存起来。你提问时，问题也被转成数字，然后系统去库里找**数字最接近**的那几段，这个过程叫**相似度检索**。

它的高明之处在于：**按"意思"找，而不是按"字面"找。** 你问"怎么把钱要回来"，它能找到写着"退款流程"的段落，哪怕两句话一个相同的字都没有。

> 📚 打个比方：向量数据库就像图书馆的**分类检索系统**。图书馆不是把书随便堆放，而是按主题分门别类；你说个大概方向，管理员就能飞快带你找到相关的那一排书。（这只是类比，实际更复杂。）

## RAG 用在哪里

RAG 是目前企业落地 AI **最实用、最普遍**的方式之一：

- **企业知识库问答**：员工问"报销流程怎么走"，AI 直接从公司制度里找答案。
- **智能客服**：基于产品手册回答用户问题，答案准确、有据可查，还能全天在线。
- **法律 / 医疗查询**：从海量条文、病例、指南中检索，辅助专业人员快速定位依据。
- **个人知识助手**：把你自己的笔记、收藏、书籍喂进去，打造"最懂你"的私人问答库。

它的最大好处：**资料一更新，答案就更新**，不用重新训练模型，还能大幅减少"瞎编"。

## RAG 和微调，怎么选？

两者不冲突，解决的问题不同：

| | 微调（第21章） | RAG（本章） |
| --- | --- | --- |
| 解决什么 | 让 AI **学会新技能、新风格** | 让 AI **用上新知识、私有资料** |
| 比喻 | 送去读研究生，改变"脑子" | 给它一本随时可查的资料书 |
| 更新知识 | 要重新训练，慢 | 改资料库即可，快 |

简单记：**要改"会不会"，用微调；要改"知不知道"，用 RAG。**

## 本章小结

- 大模型有三大短板：**知识过时、会瞎编、不懂你的私有资料**。
- **RAG = 先检索、再回答**，好比把闭卷考试变成开卷考试。
- 核心流程：**检索相关文档 → 拼进提问 → 基于资料作答**。
- **向量数据库**负责"按意思找资料"，像图书馆的分类检索系统。
- RAG 是企业落地 AI 最实用的方式，广泛用于知识库问答、客服、法律医疗等场景。
- 记住口诀：**改"会不会"用微调，改"知不知道"用 RAG。**
