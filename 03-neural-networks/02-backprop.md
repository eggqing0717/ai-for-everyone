# 第10章 神经网络如何学习：反向传播

上一章我们造好了一张神经网络，可它刚"出生"时什么都不会——里面所有的权重都是随便设的，就像一个从没学过的新手。那么问题来了：**它到底是怎么从"啥也不会"变得越来越聪明的？**

这一章，我们就来揭开这个最核心的秘密：**反向传播**。名字很吓人，但它的思路，其实和"复盘一次考试错在哪"一模一样。

## 先讲个生活场景

想象你在厨房学做一道番茄炒蛋。第一次做，你随手放盐、放糖、控制火候，端上桌一尝——太咸了！

你会怎么办？你不会把整道菜推倒重做，而是会**回想**：是不是盐放多了？糖是不是也有点影响？火候要不要调？然后你心里给每个环节"定个责"：盐的责任最大，糖次之，火候基本没问题。下一次，你重点少放盐、稍微调下糖，火候几乎不动。

炒了几十次之后，你就成了高手。

神经网络的学习，几乎就是这个过程：**做一次预测 → 尝一口发现错了多少 → 倒推每个环节该负多少责任 → 针对性地调整。** 这个"倒推追责"的过程，就是反向传播。

（这只是类比，实际的追责是靠数学计算每个权重对误差的影响，比"凭感觉"精确得多，但直觉上就是这么回事。）

## 核心原理拆解

### 1. 前向传播：先做一次预测

学习的第一步，是让数据从左到右走一遍网络。输入（比如一张图片）从输入层进去，经过一层层神经元的"加权求和 + 激活"，最后在输出层吐出一个答案，比如"我觉得这是只狗，把握 70%"。

这个"数据从左往右、一路算到底、产生一个预测"的过程，就叫**前向传播**。它就是你"先把菜炒出来端上桌"这一步。

### 2. 算损失：尝一口，错了多少

菜端上来了，得尝一口才知道好不好。网络也一样——它把自己的预测和"正确答案"对比一下，算出差距有多大。这个差距，有个专门的名字叫**损失**（loss）。

- 损失大：预测离谱，错得很惨。
- 损失小：预测很准，接近满分。

**整个学习的唯一目标，就是想尽办法把这个"损失"降到最低。** 损失就像考试的扣分，扣得越少越好。

### 3. 反向传播：像侦探一样倒推追责

现在关键来了。网络知道自己"错了 30 分"，但它有成千上万个权重，到底是哪些权重导致了错误？各自要背多大的锅？

这时候，网络就化身**侦探**。侦探破案不是从头顺着查，而是**从结果这个"案发现场"往回倒推**：最后一层的这个错误，是上一层哪几个神经元造成的？它们又是被再上一层的谁影响的？就这样一层一层往回追，把"责任"从输出层反向分摊到每一个权重身上。

这个"从结果倒推、层层追责"的过程，就是**反向传播**（back propagation）。它和前向传播方向正好相反：前向是从左到右做预测，反向是从右到左分责任。

一句话记住它：**前向传播出答案，反向传播找责任。**

### 4. 权重更新：针对性地改进

责任分清楚了，最后一步就简单了：**谁的责任大，就重点调整谁。**

- 某个权重"闯了大祸"（对错误贡献大），就多调一点。
- 某个权重基本没问题，就几乎不动。

调整的方向也很明确：朝着"让损失变小"的方向挪一小步。注意是"一小步"——一次别调太猛，不然容易矫枉过正（这一步的学问，就是上一部分讲过的"梯度下降"和"学习率"）。

然后，网络拿着更新过的权重，再来一轮：前向预测 → 算损失 → 反向追责 → 更新权重…… 如此循环成千上万次。每一轮它都错得少一点点，直到最终变成一个"炒菜高手"。

## 本章小结

- 神经网络刚出生时权重是随机的，"学习"就是不断把这些权重调好的过程。
- **前向传播**：数据从左到右走一遍网络，产生一个预测（先把菜炒出来）。
- **损失**：预测和正确答案的差距，学习的目标就是把损失降到最低（尝一口错了多少）。
- **反向传播**：从结果倒推，像侦探一样把误差的"责任"层层分摊到每个权重（复盘追责）。
- **权重更新**：责任大的多调、责任小的少调，朝减小损失的方向挪一小步。
- 这四步不断循环成千上万次，网络就越来越准。
