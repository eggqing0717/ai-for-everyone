# 第15章 自然语言处理 NLP 基础

> 我们每天张口就来的中文，在计算机眼里，其实是一堆它完全看不懂的"鬼画符"。这一章，我们就来聊聊：计算机到底是怎么开始"读懂人话"的。

## 一、先想象一个场景

假设你有一位外国朋友，他一个汉字都不认识，连"你好"两个字长什么样都分不清。现在你要教他读懂中文报纸，你会怎么做？

你大概会：先教他把句子拆成一个个词，再告诉他每个词是什么意思，最后教他理解词和词连起来的含义。

计算机学中文的过程，和教这位外国朋友几乎一模一样——只不过它比外国朋友还要"笨"一层：**它连"字形"都感受不到，它的世界里只有数字。**

这门"教计算机读懂并使用人类语言"的学问，就叫做 **自然语言处理**，英文缩写 **NLP**（Natural Language Processing）。你每天用的输入法联想、机器翻译、语音助手、垃圾邮件过滤、还有 ChatGPT，背后都是 NLP。

## 二、核心真相：计算机只懂数字，不懂文字

这是理解整个大模型世界的**第一块基石**，请一定记牢：

> **计算机的世界里没有"字",只有数字。**

在计算机底层，无论是一段文字、一张图片还是一首歌，最终都被转换成了 0 和 1 组成的数字。文字对它来说，本身没有任何"意义"，只是一串编号而已。

打个比方：**这就像图书馆里的书。** 每本书都有一个编号（比如"I247.5"），管理员靠编号找书、上架、归位，非常高效。但编号本身并不知道这本书讲的是爱情还是战争——它只是个代号。

计算机看文字也是如此：它能存储"苹果"这两个字对应的编号，却完全不知道苹果能吃、是红色的、和"香蕉"是一类东西。

**所以 NLP 要解决的核心难题就是：怎样把"文字"变成计算机能计算、而且还能保留"意思"的数字？** 这个问题，我们会用整整两章（第15、16章）来慢慢攻破。

## 三、第一步：分词——把句子切成一块块

在把文字变成数字之前，得先解决一个更基础的问题：**从哪里下刀？**

人在阅读时，会不自觉地把句子拆成一个个有意义的词。比如看到：

> 我喜欢吃苹果

你的大脑会自动切成："我 / 喜欢 / 吃 / 苹果"。这个"把连续的句子切成一个个词"的动作，就叫 **分词**（Tokenization）。

分词是 NLP 的**第一道工序**，就像做菜前要先把食材切好。切好的每一小块（可能是一个词，也可能是一个字或半个词），术语叫 **Token（词元）**。你以后一定会经常听到"Token"这个词——比如大模型按 Token 计费，说的就是这个。

### 中文分词的小麻烦

英文天生好切，因为单词之间有空格：`I like eating apples`，空格一分就完事了。

中文可就头疼了，字和字之间没有空格，还常常有**歧义**。经典的例子是：

> 南京市长江大桥

这句话到底是——
- "南京市 / 长江大桥"（南京市的那座跨长江的大桥）？
- 还是"南京 / 市长 / 江大桥"（一位姓江名大桥的南京市长）？

同样一串字，切法不同，意思天差地别。这就是为什么中文分词本身就是一门学问。（这只是类比，实际的分词算法更复杂。）

## 四、两种思路：写死规则 vs 让机器自己学

那计算机到底怎么完成分词、翻译、理解这些任务？历史上有两条截然不同的路线。

### 思路一：传统 NLP——人类把规则一条条写死

早期的做法是：**由语言专家把规则一条条写进程序里。**

比如做机器翻译，工程师就得手写成千上万条规则："遇到 apple 翻译成苹果""遇到 be 动词加 ing 表示进行时"……

这就像**编一本超级厚的规则手册**，事无巨细地规定"遇到什么情况就怎么处理"。

- ✅ **优点**：规则清清楚楚，出了错能查到是哪条规则的问题。
- ❌ **缺点**：语言太灵活了，规则永远写不完。一句"这题我不会做"和"这道菜我不会做"，"做"的含义就不同；还有各种俚语、双关、新词……人类根本没法穷举所有规则。规则手册越写越厚，还是漏洞百出。

### 思路二：神经网络——给它大量例子，让它自己学

现在的做法完全反过来了：**我们不写规则，只给计算机看海量的例子，让它自己总结规律。**

这正是我们前几部分讲过的机器学习和神经网络的思想。你不用告诉它"苹果"和"香蕉"是一类，你只要喂给它上亿句包含这些词的句子，它自己就能从"苹果和香蕉都常常出现在'吃'字后面"这类线索里，慢慢摸索出词与词的关系。

这就像**教小孩说话**：你不会拿着语法书给两岁的孩子讲"主谓宾定状补"，你只是不停地跟他说话，他听多了，自然就会说了。（这只是类比，实际的神经网络学习机制更复杂。）

下面这张表，帮你一眼看清两种思路的区别：

| 对比项 | 传统 NLP（写死规则） | 神经网络（自动学习） |
| :--- | :--- | :--- |
| 规则来源 | 人类专家手写 | 从海量数据里自己总结 |
| 灵活性 | 差，新情况就抓瞎 | 强，见多识广会举一反三 |
| 需要的数据 | 少 | 极其大量 |
| 效果 | 有天花板 | 数据越多、模型越大，效果越好 |
| 代表时代 | 上世纪至 2010 年前后 | 如今的大模型时代 |

正是"让机器自己学"这条路，最终把我们带到了大模型时代。

## 五、本章小结

- **计算机只懂数字，不懂文字**，这是理解一切的起点。NLP 的核心任务，就是把文字变成"有意义的数字"。
- **分词**是 NLP 的第一道工序，把句子切成一个个 **Token（词元）**；中文因为没有空格、常有歧义，分词更有挑战。
- 处理语言有两条路线：**传统 NLP 靠人类写死规则**（写不完、不灵活），**神经网络靠机器自己从海量例子里学**（灵活、越大越强）。
- 正是"让机器自己学"这条路，为后面的词向量、注意力、Transformer 铺平了道路。

下一章，我们就来正面攻克那个核心难题：**到底怎么把一个词，变成一串既能计算、又能保留意思的数字？** 答案就是大名鼎鼎的"词向量"。

## 六、思考题

1. 试着给"乒乓球拍卖完了"这句话找出至少两种不同的分词方式，看看意思会有什么不同。
2. 为什么说"让机器从海量数据里自己学"比"人类写死规则"更适合处理语言？你能举一个规则很难覆盖的语言现象吗（比如网络流行语、双关语）？
3. 生活中你还用过哪些 NLP 的产品？试着想想它们背后可能在做哪些"文字变数字"的工作。
