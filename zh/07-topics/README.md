# 专题篇：把 AI 放进真实问题里看

> 前面的章节帮你建立 AI 的基础地图；专题篇做另一件事：挑出当下最值得关心的问题，拆开它们在真实世界里是怎么工作的、能解决什么、又容易在哪里翻车。

## 这部分怎么读？

专题篇不是第六部分，也不是必须按顺序读的“进阶教材”。它更像一组可独立打开的放大镜：

- 想了解一个新概念，就先读“它解决什么问题”和“核心机制”。
- 想判断一个产品靠不靠谱，就重点看“适合场景”和“局限风险”。
- 想自己动手实践，就跳到“怎么设计、怎么评估、怎么落地”。

每篇专题都会尽量回答四个问题：

| 问题 | 你能得到什么 |
| --- | --- |
| 它为什么出现？ | 知道这个技术到底在补哪块短板 |
| 它怎么工作？ | 用不依赖公式的方式理解核心流程 |
| 它适合做什么？ | 能把概念连接到工作、学习和产品场景 |
| 它有什么坑？ | 不被宣传词带着跑，知道边界在哪里 |

## 建议接下来补的专题

后续可以优先补这几类，因为它们和普通读者的关系最直接：

- **多模态 AI**：为什么模型能同时看图、读文、听声音。
- **AI 搜索与 RAG 产品设计**：为什么“能查资料的 AI”仍然会答错。
- **AI 编程助手**：它到底是在补全代码，还是在参与工程协作。
- **AI 与工作流自动化**：哪些任务适合交给 AI，哪些必须留给人。
- **AI 安全与隐私**：把文件、邮件、客户数据交给 AI 前要想清楚什么。

> 读专题时不用追求“全懂”。能把一个概念放回自己的生活和工作里，知道它能做什么、不能做什么，就已经很值了。
