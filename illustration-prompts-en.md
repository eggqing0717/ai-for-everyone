# Full-Book Illustrations · Image-Generation Prompt List

> This list is extracted from all chapters of *AI for Everyone*. It gathers every **image placeholder** and its corresponding **image-generation prompt**, so you can batch-generate the images with an image AI and drop them back into the text in one pass.

## 1. Usage Notes

1. **Image placement**: All in-text images reference the `assets/images/` directory (the in-text syntax looks like `![alt text](../assets/images/filename.png)`). Simply name your generated images **strictly according to the "Target Filename" in this list**, place them in the `assets/images/` folder at the project root, and the text will display them automatically with no edits required.
2. **Filenames must match exactly**: The reference paths in the text are hard-coded. Filenames (including case, hyphens, and the `.png` suffix) must be **exactly identical** to this list, or the images will not display.
3. **Format recommendation**: Prefer **PNG** (transparent or white background is fine); for pure vector structural diagrams (such as flowcharts or architecture diagrams), **SVG** is also acceptable, but you must update the in-text reference suffix to `.svg` (this list defaults to `.png`; if you do not change the suffix, still export PNG).
4. **Consistent style**: For visual consistency, all illustrations in the book should keep the same visual language—**flat-style tech illustration, blue-purple as the main color, with English labels**—so the whole book looks unified after replacement. See "Batch Image-Generation Tips" at the end.

### Batch image-generation tips

1. **Consistent art style**: The whole book uses a **flat-style tech illustration** look, avoiding photorealistic or 3D rendering (a few "3D valley" / "boiling water" cases can be handled flexibly per the prompt, but keep the overall lines and texture consistent).
2. **Consistent color scheme**: Use **blue-purple as the main color**; for comparison-type images, use "green = positive/opportunity, orange-red = negative/risk" as accent colors; the two appendix images already provide precise color values that can serve as a book-wide palette reference—tech blue `#2E7BF6`, teal `#22C1A6`, orange highlight `#FF8A3D`, light gray-white background.
3. **English labels**: All in-image text **must use English** (English terms such as GPT, CNN, Q/K/V, Σ may be kept); use a clear font at a moderate size to ensure it stays readable when scaled down.
4. **Resolution recommendation**: For in-text horizontal images, use **1600×900 or 1920×1080 (16:9)**; for square/heatmap types, use **1200×1200 (1:1)**; export **PNG (72–150 DPI is enough for web display)**. For structural/flow diagrams, prefer **SVG** to keep scaling crisp (remember to update the in-text reference suffix accordingly).
5. **Names must match the table**: After generating, rename each file one by one according to the "Target Filename" in this list, place them all in `assets/images/`, and you can seamlessly replace the book's placeholder images.
