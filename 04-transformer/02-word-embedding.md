# 第16章 词向量：把文字变成数字

> 上一章我们留了个大问题：怎么把一个词变成"有意义的数字"？这一章的主角——**词向量**，就是人类想出的绝妙答案。它是整个大模型世界里，我个人觉得最优雅、最"聪明"的一个想法。

## 一、先从一个笨办法说起

我们已经知道，计算机只认数字。那最简单粗暴的做法是什么？

**给每个词编个号呗！** 比如：

> 苹果 = 1，香蕉 = 2，国王 = 3，女王 = 4，汽车 = 5……

看起来没问题？但麻烦大了。因为数字之间是有大小关系的，计算机会误以为：香蕉(2) 比苹果(1) "大一倍"，国王(3) 加香蕉(2) 等于女王(4)+苹果(1)……这些计算完全是胡来的，编号只是代号，根本不该拿来加减。

于是人们想了个改进办法，叫 **One-Hot 编码（独热编码）**。

## 二、One-Hot 编码：一面面互不相干的旗子

One-Hot 的思路是：**假设词典里一共有 1 万个词，那就给每个词发一个长度为 1 万的"数字串"，其中只有它自己的位置是 1，其余全是 0。**

想象一个**巨大的开关面板，上面有 1 万个开关**。每个词只负责按下属于自己的那一个开关，其余全部保持关闭：

| 词 | 它的 One-Hot 表示（示意） |
| :--- | :--- |
| 苹果 | [1, 0, 0, 0, 0, ……] |
| 香蕉 | [0, 1, 0, 0, 0, ……] |
| 国王 | [0, 0, 1, 0, 0, ……] |
| 女王 | [0, 0, 0, 1, 0, ……] |

这样一来，计算机就不会误以为词之间有大小关系了。每个词都是一面**独立的旗子**，谁也不挨着谁。

但是——**这个办法有两个致命毛病：**

1. **太浪费。** 词典有多少个词，每个词的数字串就有多长。几万个词，就是几万个数字，其中 9999 个都是 0，太浪费空间了。
2. **词与词之间毫无关系。** 这才是最要命的。在 One-Hot 眼里，"苹果"和"香蕉"的距离，跟"苹果"和"坦克"的距离，是**一模一样远**的。可我们人类都知道，苹果和香蕉明明是好兄弟（都是水果），跟坦克八竿子打不着啊！

One-Hot 就像一群**互不相识的陌生人**，每个人都举着自己的旗子，谁也不认识谁，完全体现不出"苹果和香蕉是一类"这种宝贵的意思。

## 三、词向量登场：把词变成"坐标点"

科学家们于是想：能不能让"意思相近的词"，在数字上也"离得近"呢？

**这就是词向量（Word Embedding）的核心思想：把每一个词，变成一个多维空间里的"坐标点"。意思越接近的词，坐标点离得越近。**

先别被"多维空间"吓到，我们从最简单的**二维坐标系**（就是初中学过的那个 X-Y 平面）说起。

假设我们只用两个维度来描述词，比如横轴代表"是不是食物"，纵轴代表"是不是交通工具"。那么：

- "苹果""香蕉"会挤在**右下角**（很像食物，不像交通工具）；
- "汽车""飞机"会跑到**左上角**（很像交通工具，不像食物）；
- 苹果和香蕉这两个点，自然就**离得很近**；苹果和汽车，就**离得很远**。

![词向量：意思相近的词在坐标系里靠得近](../assets/images/ch16-word-vector-space.png)

> 🎨 生图提示词：一张二维坐标系示意图，教学插画风格，中文标注。横轴标注"食物程度"，纵轴标注"交通工具程度"。在右下方聚集一簇标着"苹果、香蕉、西瓜"的彩色小圆点并圈在一起；左上方聚集"汽车、飞机、火车"的小圆点圈在一起；相近的点用虚线连接表示距离近。配色清新，风格简洁友好，突出"相近的词聚在一起"的直觉。

**这样一来，神奇的事情发生了：** 词与词的"意思相似度"，变成了坐标点之间"距离的远近"。计算机只要算一算两个点离得近不近，就能判断两个词意思像不像了！

当然，真实世界里两个维度远远不够。实际的词向量往往有**几百甚至上千个维度**，能同时描述一个词在"食物、情感、性别、大小、褒贬……"等无数方面的特征。维度太多我们脑子想象不出来，但计算机算起来毫无压力。（这只是类比，实际维度的含义并不像"食物程度"这么清晰，而是模型自己学出来的、人类未必能解读的抽象特征。）

## 四、见证奇迹：国王 - 男人 + 女人 ≈ 女王

词向量最让人拍案叫绝的地方来了。既然每个词都成了坐标点（也就是一串数字），那我们就能**对它们做加减法**。而算出来的结果，竟然真的符合语义！

最著名的例子是：

> **国王 − 男人 + 女人 ≈ 女王**

这个式子在说什么？用大白话翻译一下：

- "国王"减掉"男人"，剩下的是一种"王室、掌权者"的抽象含义；
- 再加上"女人"，就得到了"女性的王室掌权者"；
- 而最接近这个结果的词，正是——**女王**！

是不是有点起鸡皮疙瘩？计算机明明不"懂"什么是国王女王，但通过词向量的加减，它居然"算"出了人类才懂的语义关系。类似的还有：

> **北京 − 中国 + 法国 ≈ 巴黎**（首都与国家的关系）
> **走 − 现在 + 过去 ≈ 走过**（时态的关系，示意）

这说明：**词向量不只记住了词，还悄悄记住了词与词之间的"关系"，并且这种关系可以像数学一样运算。** 语言里那些微妙的规律（性别、首都、时态……），被巧妙地"藏"进了坐标的方向里。（这只是类比，实际运算结果是"最接近"而非精确相等。）

![词向量的类比运算：国王-男人+女人≈女王](../assets/images/ch16-king-queen-analogy.png)

## 五、这些词向量是怎么来的？——预训练思想

你可能会问：这么神奇的坐标，是谁一个个标出来的？总不能靠人手工给几万个词打分吧？

**当然不是。这些坐标，是计算机自己"读书读出来的"。** 这里就引出一个极其重要的思想——**预训练（Pre-training）**。

方法的核心是一句古老的语言学直觉：

> **"观其伴，知其义"——想知道一个词什么意思，看看它周围常和哪些词一起出现就行了。**

比如你从没见过"雺"这个字，但如果你反复读到"今天早上起了好大的雺，能见度很低""雺散了，太阳出来了"，你立刻就能猜到：它八成是"雾"一类的东西。

计算机也用同样的办法：**把互联网上的海量文本（新闻、小说、维基百科……）全都读一遍，统计每个词都和哪些词做邻居**，然后不断微调每个词的坐标，让"经常做邻居的词"坐标靠近，"从不相干的词"坐标远离。读的书足够多，每个词的坐标就自然而然地"排布"到了合理的位置。

这个"提前读遍海量文本、把词向量学好"的过程，就叫**预训练**。它有几个了不起的好处：

- **一次学习，反复使用。** 词向量学好之后，可以拿去做翻译、做问答、做情感分析……各种任务都能用，不用每次从零开始。
- **省时省力。** 就像一个博览群书的人，无论你让他干哪一行，他都有基础、上手快。

**记住"预训练"这个词。** 后面 GPT 的名字里那个 **P**，正是 Pre-training（预训练）的缩写。今天所有大模型"先读遍全网、再学具体任务"的套路，思想源头就在这里。

## 六、本章小结

- 给词编号、One-Hot 编码都有硬伤：**要么误导计算机，要么词与词毫无关系**。One-Hot 就像一群互不相识、各举旗子的陌生人。
- **词向量**把每个词变成多维空间里的一个**坐标点**，让"意思相近的词，坐标离得近"。相似度问题，变成了距离问题。
- 词向量能做加减法，还符合语义，比如经典的 **"国王 − 男人 + 女人 ≈ 女王"**，说明它悄悄学会了词与词之间的关系。
- 这些坐标不是人工标的，而是靠**预训练**——让模型"读遍海量文本、观其伴知其义"自己学出来的。**预训练**是通往大模型的关键思想。

我们现在已经能把词变成漂亮的坐标点了。可是，光有一个个孤立的词还不够——一句话里，哪个词更重要？词和词之间怎么互相影响？下一章的 **注意力机制**，就要来解决这个问题。

## 七、思考题

1. 用你自己的话解释：为什么说 One-Hot 编码"体现不出词与词的关系"，而词向量可以？
2. "国王 − 男人 + 女人 ≈ 女王"这个例子里，被"减掉又加上"的，你觉得是词的哪一部分含义？试着自己编一个类似的类比式子。
3. "预训练"为什么能省时省力？把它类比成一个人的成长经历，你会怎么描述？
